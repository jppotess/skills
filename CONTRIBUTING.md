# Contributing

Agent Delivery Kit is intentionally small. Contributions should preserve the public delivery bundle and keep the skills portable across projects.

## Before Opening A Change

- Keep `codex/skills` as the source of truth.
- Keep reusable skill bodies free of secrets, local machine paths, private repo URLs, and company-specific policy.
- Put team-specific workflow details in repo-local docs generated by `$delivery-kit-setup`, not in global skills.
- If a skill is added, renamed, or removed, update:
  - `.claude-plugin/plugin.json`
  - `README.md`
  - `docs/skill-reference.md`
  - `setup.sh`
  - `install.sh`
  - `scripts/validate-skills.sh`

## Validate

Run:

```bash
scripts/validate-skills.sh
npx skills@latest add jppotess/skills -l
./setup.sh --agent both --dry-run
./install.sh --dry-run
```

For installer changes, also test a temporary destination:

```bash
tmpdir="$(mktemp -d)"
./install.sh --dest "$tmpdir"
find "$tmpdir" -maxdepth 2 -name SKILL.md | sort
rm -rf "$tmpdir"
```

## Skill Naming

Use short, invocation-friendly names:

- lower-case
- hyphen-separated
- memorable enough to type after `$`
- descriptive enough to make routing obvious

Prefer docs and examples that explain when to use a skill, what evidence it should produce, and what it must not claim.
